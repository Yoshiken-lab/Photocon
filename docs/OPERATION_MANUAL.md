## 2. 認証機能 (Authentication) 運用方針

### 🎯 採用戦略: 「段階的ローンチ」
**(2026-01-29 決定)**

**背景**:
- 認証機能には `NEXT_PUBLIC_ENABLE_AUTH` フラグで ON/OFF を切り替える仕組みがある。
- しかし、ON/OFF を頻繁に切り替えると、ユーザーが「マイページで自分の投稿が見えない」等の混乱を招くリスクがある。

**運用方針**:
1.  **フェーズ1: テスト運用（Auth OFF）**
    - `NEXT_PUBLIC_ENABLE_AUTH=false` で運用開始。
    - 数回のイベントを開催し、システムの安定稼働を確認する。
    - この期間の投稿は `user_id` が記録されず、後からマイページで確認できない（仕様通り）。

2.  **フェーズ2: 本格ローンチ（Auth ON 恒久化）**
    - 運営が慣れてきたタイミングで、**「マイページ機能新登場！」** と告知。
    - `NEXT_PUBLIC_ENABLE_AUTH=true` に **恒久的に** 切り替える。
    - この時点以降の投稿は全て `user_id` が紐付き、ユーザーはマイページで自分の投稿を管理できる。

**ポイント**:
- 「新機能が追加された」という見せ方をすることで、過去の投稿が見えなくても不信感を持たれない。
- **一度 Auth ON にした後は、OFF に戻さない** ことを推奨。

## 3. アカウント削除申請の処理手順

**(2026-01-29 実装)**

### 📋 概要
ユーザーがマイページから「アカウント削除申請」を行うと、運営が承認/却下を行う。
承認すると、ユーザーの認証情報が削除され、ログインできなくなる。
**投稿した写真はギャラリーに残る（`user_id = NULL` に更新される）。**

### 🔧 処理フロー

1. **ユーザー**がマイページ下部「アカウント設定」の「退会手続きへ進む」をクリック
2. 理由を入力して送信（理由は任意）
3. **運営**が管理画面 `/admin/account-deletions` で申請を確認
4. 「承認（削除実行）」または「却下」を選択
5. 承認すると、ユーザーのSupabase認証情報が削除される
6. **手動でメール送信**: 管理画面のメールアドレス横「コピー」ボタンでアドレスをコピーし、削除完了メールを手動で送信する

### 📧 削除完了メール文例

```
件名: 【スクールフォト！フォトコンテスト】アカウント削除完了のお知らせ

いつもご利用ありがとうございました。

ご依頼いただいたアカウントの削除が完了いたしました。

今後ともスクールフォト！をよろしくお願いいたします。

スクールフォト！フォトコンテスト事業部
```
