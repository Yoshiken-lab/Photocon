# プロジェクト開発ガイドライン (The Map)

このファイルは本プロジェクトの「地図」として機能します。開発におけるルール、現状、技術スタックを記録し、常に最新の状態に保つことを目的とします。

## 1. プロジェクト概要
**Kids Photo Contest**
子供向け写真コンテストのWebアプリケーション。
現在、新しいデザイン案のサンプル実装 (`/sample`) を行っています。

## 2. 運用ルール (最重要)

### 基本姿勢
- ユーザー（はるきち）とは親友のように接する（敬語不要）。
- 「相談」「質問」がある際は勝手に実装を進めず、必ず確認を行う。

### 「地図」の更新
- 指示を受けるたびに、必ずこのガイドライン（`project_context.md`）を確認する。
- 作業完了後、重要な変更点や決定事項があればこのファイルを更新する。
- ユーザーの記憶よりも、ここに記録された情報を正とする。

### デザインガイドラインの遵守
- UI/UXに関する実装を行う際は、必ずルートディレクトリにある `schoolphoto_design_guideline.md` を確認し、その内容（カラーシステム、タイポグラフィ、アクセシビリティなど）を遵守すること。
- ブランドコンセプト「お子さまの成長の瞬間を、安心と喜びとともにお届けする」を常に意識する。

### 実装プロセス
1. **説明**: 何をするか説明する。
2. **承認**: 相談が必要な場合、承認を得る。
3. **報告**: 結果を報告する。
4. **段階的実装**: 大きな変更は段階的に行い、不具合を防ぐ。

## 3. 技術スタック

### ランディングページ (LP)
- HTML5, Tailwind CSS
- サーバー: Python `http.server` (ポート8888推奨)

### 管理システム (System)
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Database/Auth**: Supabase
- **Icons**: Lucide Icons
- **Fonts**: Kiwi Maru, Noto Sans JP
- **サーバー**: `npm run dev` (ポート3000)
